Build now Minimal database only Draft: 2026-05-22

Small enough to build, clear enough for the team.

This visual contract turns the MVP API into a quick reference for customer QR sessions, chat, staff takeover, and service requests. Menu, cart, order, review, and analytics are intentionally marked as planned work.

v0.1 focus Restaurant table communication through chat.
Not real ordering yet `order_request` is a chat/request type only.
Staff protected Staff identity comes from auth, not request body.

Minimal User Flow

1 QR scanCustomer opens table link.
2 Session startsBackend creates anonymous session.
3 Chat runsCustomer, AI, staff, and system messages persist.
4 Request createdWater, bill, help, or order-related need.
5 Staff resolvesDashboard tracks status and active sessions.

MVP Database Scope

Only these six tables are assumed by API Contract v0.1.

restaurantsBasic restaurant profile.
restaurant_tablesTable numbers and QR tokens.
customer_sessionsAnonymous dine-in visits.
staff_usersStaff and manager logins.
chat_messagesCustomer, AI, staff, and system chat.
customer_requestsConcrete service tasks for staff.
